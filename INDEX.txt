🔑 CONCEPTOS CLAVE (Resumen)
──────────────────────────────

SESGO (BIAS)
├─ Definición: Error por asumir simplificaciones
├─ Alto sesgo → Modelo demasiado simple
├─ Síntoma: Error alto en train Y test (similar)
└─ Solución: Aumentar complejidad del modelo

VARIANZA (VARIANCE)
├─ Definición: Sensibilidad a fluctuaciones
├─ Alta varianza → Modelo demasiado complejo
├─ Síntoma: Error bajo en train, ALTO en test
└─ Solución: Reducir complejidad o agregar datos

TRADE-OFF
├─ Error Total = Sesgo² + Varianza + Ruido
├─ No puedes minimizar ambos simultáneamente
├─ Existe un punto óptimo (balance)
└─ Objetivo: Encontrarlo

CON DATOS LIMITADOS (28 observaciones)
├─ Overfitting es muy probable
├─ Modelos simples funcionan mejor
├─ Regularización es CRÍTICA
└─ Validación cruzada es OBLIGATORIA

🎓 LECCIONES APRENDIDAS
────────────────────────

1. ✅ Simplicidad gana con datos limitados
   → Regresión lineal > Polinomio grado 5 con 28 datos

2. ✅ La brecha train-test es una métrica crítica
   → Si brecha > 100: overfitting severo
   → Si brecha < 20: buena generalización

3. ✅ Un modelo perfecto en train es sospechoso
   → MSE_train = 0 usualmente significa memorización
   → Verificar SIEMPRE en conjunto de prueba

4. ✅ Validación es obligatoria con datos pequeños
   → Never confíes solo en métricas de train
   → Siempre evalúa en prueba

5. ✅ El trade-off sesgo-varianza es inevitable
   → No puedes minimizar ambos
   → Busca balance, no perfección
